# 【核心知识框架】  

```shell
计算机网络概述
├── 通信模型
│   ├── 调制解调器 (Modem)  # 调制器 (Modulator) 和解调器 (Demodulator) 的缩写，用于数字信号与模拟信号之间的转换
│   ├── C/S 模型 (Client-Server Model)  # 客户端-服务器模型，用于信息访问
│   └── P2P 通信 (Peer-to-Peer Communication)  # 对等通信模型
├── 网络构成要素
│   ├── 节点 (Node)  # 包括计算机、交换机、路由器等
│   ├── 链路 (Link)  # 连接节点的逻辑线路，物理上可以是线缆或无线电波
│   ├── 主机 (Host)  # 提供服务的计算机，也称服务器
│   └── 客户端 (Client)  # 接收服务的计算机，也称终端
├── 网络分类
│   ├── 按分布范围分类
│   │   ├── 个域网 (PAN)  # 一般范围在几米，如个人设备之间
│   │   ├── 局域网 (LAN)  # 范围在1km以内，如家庭、公司、校园
│   │   ├── 城域网 (MAN)  # 范围在5km~50km，如一个城市
│   │   └── 广域网 (WAN)  # 范围数百至数千千米，如国家或国际网络
│   ├── 按用途/结构分类
│   │   ├── 公用网 vs 专用网
│   │   ├── 电路交换 vs 报文交换 vs 分组交换
│   │   ├── 星型、总线型、环型、树型、网状型 (拓扑结构)
│   │   └── 有线 (双绞线、同轴电缆、光纤) vs 无线
│   └── 按网络使用者分类
│       └── 教育、科研、商业、企业、政府等
├── 性能指标
│   ├── 时延 (Delay) # 【装】->【走】->【处理】->等】
│   │   ├── 发送时延【装】 (Transmission Delay)  # 数据帧发送所需时间
│   │   ├── 传播时延【走】 (Propagation Delay)  # 电磁波在信道中传播时间
│   │   ├── 处理时延【处理】 (Processing Delay)  # 路由器处理数据所需时间
│   │   └── 排队时延【等】 (Queuing Delay)  # 数据在路由器队列中等待时间
│   ├── 带宽 (Bandwidth)  # 单位时间内信道传输的最高数据率，单位为 bit/s
│   ├── 速率 (Rate)  # 传输速率，单位为 bit/s 或 byte/s
│   ├── 时延带宽积 (Delay Bandwidth Product)  # 传播时延和带宽的乘积，表示路径上数据量
│   └── 往返时间 (RTT)  # 从发送端发送数据到接收端确认返回的时间
├── 报文与分组
│   ├── 报文 (Message)  # 网络传输的数据单元
│   └── 分组 (Packet)  # 报文被分割成更小的数据块，包含首部信息
└── ISP 层次结构
    ├── 主干 ISP
    ├── 地区 ISP
    └── 本地 ISP
```

# 【公式与计算点】  

 🎯 四种时延，一句话解释 👉 **装 → 走 → 处理 → 等**
 
- **【装】发送时延**：把数据“推上网线”的时间
- **【走】传播时延**：信号在链路上“跑”的时间
- **【处理】处理时延**：路由器（互联网骨干/运营商设备）“看一眼”的时间
- **【等】排队时延**：在路由器“等处理”的时间

1. **发送时延**  【“装”多久】与数据大小有关   $$
   \text{发送时延}(s) = \frac{\text{数据帧长度 (bit)}}{\text{发送速率 (bit/s)}}
   $$
2. **传播时延**  【“走”多久】与距离有关
   $$
   \text{传播时延}(s) = \frac{\text{信道长度 (m)}}{\text{电磁波传播速度 (m/s)}}
   $$
3. **时延带宽积**  【正在“走”的数据】$$
   \text{时延带宽积}(bit) = \text{传播时延} \times \text{带宽}
   $$
    👉 单位：bit，表示在传播时延期间信道上能传输的数据量。
4. 往返时间RTT【去程 + 回程】
$$RTT = 数据往返一次的总时延 = 2 ×（发送时延 + 传播时延 + 处理时延 + 排队时延）$$
    👉 即：**去程 + 回程**

# 易混考点
## 速率/存储 单位换算

> 
> **在网络传输中**，1Mb/s = 1000Kb/s；**在存储中**，1MB = 1024KB。
> 

## 带宽和速率

> 
> 👉 **带宽 ≈ 理论最大传输能力（上限）**  
> 👉 **速率 ≈ 实际传输速率（当前速度）**
>

# 📌 **总结复习提示**：  
- 网络分类是高频考点，尤其是 LAN、WAN、MAN 的区别。  
- 性能指标中的时延、带宽、速率是必须掌握的计算点。  
- 报文与分组、C/S 与 P2P 模型是易混淆点，需要重点区分。  
- 单位换算是考试中常见的陷阱题，务必注意网络与存储的差异。

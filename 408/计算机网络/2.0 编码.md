【所属章节/主题】  
物理层（传输技术与编码方式）

【核心知识框架】  
```shell
# 物理层 - 编码方式
├── 电平编码 (Level Encoding)  
│   ├── 归零编码 (RZ - Return to Zero)  
│   │   └── 逻辑1: 正电平; 逻辑0: 负电平; 每位传输后返回零电平  
│   │   └── 三态: 正电平｜零电平｜负电平  
│   │   └── 优点: 信号有明显过渡，便于同步  
│   │   └── 缺点: 高频分量多，传输距离短  
│   └── 不归零编码 (NRZ - Non-Return to Zero)  
│       └── 逻辑1: 正电平; 逻辑0: 负电平; 信号不归零  
│       └── 两态: 正电平｜负电平  
│       └── 优点: 频率低，适合远距离传输  
│       └── 缺点: 无时钟同步，连续0/1难区分  
│       └── 变体: 反向不归零编码 (NRZI - Non-Return to Zero Inverted)  
│           └── 信号翻转表示0，不翻转表示1  
│           └── 优点: 改善同步能力，减少直流分量  
│           └── 缺点: 仍需时钟同步  
├── 曼彻斯特编码 (Manchester Encoding)  
│   └── 特征: 每位中间有跳变  
│       ├── 上升沿: 逻辑0  
│       ├── 下降沿: 逻辑1  
│       └── 优点: 自同步，无直流分量  
│       └── 缺点: 传输效率低（50%）  
│       └── 应用: 以太网 10BASE5/10BASE2  
└── 差分曼彻斯特编码 (Differential Manchester Encoding)  
    └── 特征: 每位开始有跳变  
        ├── 翻转: 逻辑0  
        ├── 不翻转: 逻辑1  
        └── 优点: 抗干扰能力强，适用于噪声环境  
        └── 缺点: 传输效率低（50%）  
        └── 应用: 以太网 10BASE-T  
```

【公式与计算点】  
- **4B/5B编码**：  
  - 用5位编码表示4位数据，增加冗余位，提升传输效率和同步能力。  
  - 编码效率：$ \frac{4}{5} = 80\% $，即20%的冗余。  
  - 出题点：4B/5B编码表的对应关系，例如：  
    - 4位二进制（0000）→ 5位编码（10010）  
    - 4位二进制（0001）→ 5位编码（10011）  
    - 以此类推（原文中“打曲阑千4Bi5B 码”疑似OCR错误，应为“4B/5B编码表”）

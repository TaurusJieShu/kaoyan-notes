## 📦 数组（Array）
### 标准定义

```c
#include <stdio.h>

#define MAXSIZE 100

typedef int ElemType;

typedef struct {
    ElemType data[MAXSIZE];  // 相当于 int data[100];
    int length;
} SeqList;

// 初始化顺序表
void init_list(SeqList *L)
{
    L->length = 0;
}

int main()
{
    SeqList L;
    init_list(&L);
    ...
    return 0;
}
```

```text
【1】数据本质：一组连续存储的同类型数据
【2】关系结构：线性顺序关系（通过下标隐式连接）
【3】典型特征：有序、连续、随机访问快
【4】适用问题
- 需要快速访问（O(1)）
- 数据规模固定或变化不频繁
  
【5】实现方式：连续内存空间
【6】核心操作
- 下标访问
- 遍历
- 查找（顺序 / 二分）

【7】不适用场景
- 频繁插入删除
- 动态扩展需求强

【8】一句话总结：用“位置”表示关系的数据结构
```

## 🔗 链表（Linked List）

```text
【1】数据本质
离散存储的数据集合

【2】关系结构
通过指针连接的线性关系

【3】典型特征
不连续、动态、顺序访问

【4】适用问题
- 频繁插入删除
- 不关心随机访问

【5】实现方式
节点 + 指针

【6】核心操作
- 指针遍历
- 插入 / 删除
- 反转

【7】不适用场景
- 需要快速定位第k个元素

【8】一句话总结
用“指针”显式表示关系的数据结构
```

---

## 🌳 树（Tree / 二叉树）

```text
【1】数据本质
具有层级关系的数据

【2】关系结构
父子关系（1对多）

【3】典型特征
层级结构、递归定义

【4】适用问题
- 表达层级（组织结构 / 表达式）
- 分治问题

【5】实现方式
- 链式（最常见）
- 顺序（完全二叉树用数组）

【6】核心操作
- 递归遍历（前中后序）
- 层序遍历
- 求高度 / 节点数

【7】不适用场景
- 复杂网络关系（非层级）

【8】一句话总结
用“层级关系”组织数据的结构
```

---

## 🕸 图（Graph）

```text
【1】数据本质
任意元素之间存在关系的数据

【2】关系结构
多对多关系（网络结构）

【3】典型特征
无层级、关系复杂、可有向/无向

【4】适用问题
- 网络关系（社交 / 路径）
- 连通性 / 最短路径

【5】实现方式
- 邻接矩阵（稠密图）
- 邻接表（稀疏图，数组+链表）

【6】核心操作
- BFS / DFS
- 连通分量
- 路径搜索

【7】不适用场景
- 简单线性或层级问题

【8】一句话总结
用“任意连接关系”描述数据的结构
```

---

## 📚 栈（Stack）

```text
【1】数据本质
受限的线性结构

【2】关系结构
后进先出（LIFO）

【3】典型特征
只能一端操作

【4】适用问题
- 递归模拟
- 括号匹配
- 表达式计算

【5】实现方式
- 数组
- 链表

【6】核心操作
- push / pop

【7】不适用场景
- 随机访问需求

【8】一句话总结
限制访问顺序的线性结构
```

---

## 🚶 队列（Queue）

```text
【1】数据本质
受限的线性结构

【2】关系结构
先进先出（FIFO）

【3】典型特征
一端进，一端出

【4】适用问题
- 层序遍历
- BFS

【5】实现方式
- 数组（循环队列）
- 链表

【6】核心操作
- enqueue / dequeue

【7】不适用场景
- 需要回溯或逆序处理

【8】一句话总结
按时间顺序处理数据的结构
```

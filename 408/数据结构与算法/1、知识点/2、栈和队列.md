### 结构
- **栈（Stack）**     ：**后进先出（LIFO）**，**数组** 或 **链表**实现
- **队列（Queue）**：**先进先出（FIFO）**，**循环队列（数组）** 或 **链表**。
### 对比
- 栈适用场景：需要后进先出逻辑（如递归、括号匹配）
- 队列适用场景：需要先进先出逻辑（如任务调度、缓冲区）
### 方法
栈相关
1. 栈：匹配、嵌套关系、匹配消除
2. 双栈：需要保留前后最近的关系，且有栈特征（浏览器的前进后退）
3. 单调栈：找最大/小值，删除某个序列中，最大和最小的k个内容（找当前最大值）

队列相关：
1. 队列：适合层序遍历，处理同级关系，先后到达关系
2. 队列（循环队列）：任务调度，先来先服务
3. 双端队列（单调队列）：窗口内元素的最大值

其他：
1. 哈希集合/哈希表：是否出现过，存在性问题。【数组/链表】
2. BFS/DFS：BFS更适合找最短路径；DFS更适合找全部路径【栈/队列】

# 多维数组&稀疏矩阵
## 常见的多维数组

```text
二维数组：
[[1, 2, 3],
 [4, 5, 6],
 [7, 8, 9]]

二/三维数组访问方式：
a[i][j]
a[x][y][z]
```

常见用途：
- 矩阵运算
- 棋盘、地图、迷宫
- 动态规划表
- 图像像素存储
- 邻接矩阵表示图

多维数组在内存中也是线性存储：
1. 按行优先（Row-major）
2. 按列优先（Column-major）

刷题中常见的多维数组场景：
1. 网格类问题：岛屿数量、矩阵最短路径、旋转矩阵、搜索二维矩阵
2. 动态规划：`dp[i][j]`、`dp[i][j][k]`
3. 图的表示：邻接矩阵 `graph[i][j]`

## 稀疏矩阵&稠密矩阵

- **稀疏矩阵（Sparse Matrix）**：  矩阵中**大多数元素都是 0**，只有少量元素非 0
- **稠密矩阵（Dense Matrix）**：非零元素较多

对稀疏矩阵压缩的特点：
优点：
- 节省存储空间
- 某些运算更高效
- 更适合表示“关系很少”的大规模数据
缺点：
- 随机访问可能不如普通二维数组方便
- 实现更复杂
- 某些操作需要额外索引结构

### 特殊矩阵
1. **对称矩阵**
	1. 公式：$a_{ij} = a_{ji}$ 即 $a[i][j] = a[j][i]$ 
	2. 存储数量：$\frac{n(n+1)}{2}$ （$n$行$n$列，只需存上三角或下三角）
	3. 偏移计算：根据上下三角矩阵计算
2. **上三角矩阵**
	1. 公式：$a_{ij} = 0 \quad (i > j)$
	2. 存储数量：$\frac{n(n+1)}{2}$（第$0$行有$n$个，第$1$行有$n-1$个，...，第$n-1$行有$1$个）
	3. 行优先压缩存储下标计算（当$i \le j$时）：
		1. 前$i$行元素总数：$\frac{i(2n-i+1)}{2}$
		2. 当前行内偏移量：$j-i$
		3. 所以下标为：$\frac{i(2n-i+1)}{2} + (j-i)$
3. **下三角矩阵**
	1. 公式：$a_{ij} = 0 \quad (i < j)$
	2. 存储数量：$\frac{n(n+1)}{2}$（第$0$行有$1$个，第$1$行有$2$个，...，第$n-1$行有$n$个）
	3. 行优先压缩存储下标计算（当$i \ge j$时）：
		1. 前$i$行元素总数：$\frac{i(i+1)}{2}$
		2. 当前行内偏移量：$j$
		3. 所以下标为：$\frac{i(i+1)}{2} + j$
	```c
	// 上三角矩阵
	int get_index(int i, int j, int n) {
	    if (i > j) {   // 对称矩阵，转到上三角
	        int temp = i;
	        i = j;
	        j = temp;
	    }
	    return i * (2 * n - i + 1) / 2 + (j - i);
	}
	// 下三角矩阵
	int get_index(int i, int j, int n) {
	    if (i < j) {   // 对称矩阵，转到上三角
	        int temp = i;
	        i = j;
	        j = temp;
	    }
	    return i(i - 1) / 2 + j
	}
	```
4. **对角矩阵**
	1. 公式：$a_{ij} = 0 \quad (i \ne j)$，除主对角线外，其余元素全为 $0$
	2. 存储数量：$n$（只有主对角线上的$n$个元素可能非零）
	3. 压缩存储下标计算：
	```c
	// 主对角线
	int get_by_index(int *arr, int i, int j) {
		    if (i != j) return 0;
		    else return arr[i];
	}
	// 副对角线
	if (i + j + 1 != n) return 0;
	else return arr[i];
	```
5. **带状矩阵**
	1. 公式：$a_{ij} = 0 \quad (|i-j| > 1)$
	2. 存储数量：$3n-2$
		1. 主对角线有$n$个
		2. 上对角线有$n-1$个
		3. 下对角线有$n-1$个
	3. 特点：
		1. 只有主对角线、主对角线上方一条、主对角线下方一条可能非零
		2. 其余元素全为$0$
	4. 压缩存储方式：
		1. 可按行优先存储这$3n-2$个元素
		2. 也可看作带状矩阵的一种特殊情况
	5. 带状矩阵说明：
		1. 若一般带状矩阵满足$|i-j| \le k$时元素可能非零，否则为$0$
		2. 则三对角矩阵就是$k=1$时的特殊带状矩阵
	6. 行优先压缩存储下标计算（三对角矩阵）：
		1. 当$|i-j| > 1$时，$a_{ij} = 0$
		2. 当$|i-j| \le 1$时，可用相对位置顺序压缩存储，常见总元素数为$3n-2$
可以，下面直接给你一个**带状矩阵示意图**。

5. **带状矩阵（条带状的矩阵）**
	1. 公式：$a_{ij} = 0 \quad (|i-j| > k)$
	2. 含义：
		1. 主对角线附近一条“带”内元素可能非零，其余位置为$0$。
		2. k = 1 时被称为三对角矩阵
		3. 以主对角线为中心，向上保留$k$条对角线，向下保留$k$条对角线，其余位置全部压成$0$
	```c
	// 三对角矩阵 k = 1 |i-j| <= 1
	x x 0 0 0
	x x x 0 0
	0 x x x 0
	0 0 x x x
	0 0 0 x x
	 
	// 五对角带状矩阵 k = 2 |i-j| <= 2
	x x x 0 0 0
	x x x x 0 0
	x x x x x 0
	0 x x x x x j = 1 i = 3 k = 2 n = 6 sum += 0
	0 0 x x x x j = 2 i = 4 k = 2 n = 6 sum += 0
	0 0 0 x x x
	```
	
	```c
	// 带状矩阵
	int get_by_index(int *arr, int i, int j, int k, int n) {
		int d = i - j;
		if (d < 0) d *= -1;
		if (d > k) return 0;
		int sum = 0;
		for (int col = 0; col < i; col ++){
			if(col < k) sum += (1 + k + col)
			else if (col >= n - k) sum += (k + n - col)
			else sum += (2 * k + 1)
		}
		
		int left; 
		if (i < k) left = 0; 
		else left = i - k; 
		
		return arr[sum + (j - left)];
	}
	```


### **稀疏矩阵**常见的表示方法
#### 三元组表示法
只存储非0元素的内容
```text
0 0 0        可以表示为：
0 5 0        → (1,1,5)
7 0 0        → (2,0,7)
```

如果再加上矩阵规模信息：

```text
rows = 3, cols = 3, count = 2
```

这是最基础的稀疏矩阵表示法。
用途：
- 教材中的基本表示
- 理解稀疏矩阵存储思想
- 适合非零元素遍历

#### 压缩行 CSR / 列存储 CSC
按 **行 / 列** 去看矩阵，遇到非零元素才存。

通常用 3 个数组：
- `values`：存所有非零值
- `col_index / row_index`：这些非零值分别在第几 **列 / 行**
- `row_ptr / col_ptr`：每一 **行 / 列** 的非零元素在 `values` 里从哪里开始
矩阵：
```text
1 0 0 2
0 0 3 0
4 0 5 0
```
所以，以行压缩举例：
```text
values    = [1, 2, 3, 4, 5] // 值
col_index = [0, 3, 2, 0, 2] // 列号
row_ptr   = [0, 2, 3, 5]    // 
```
例如：
- 第 0 行：`[0, 2)`
- 第 1 行：`[2, 3)`
- 第 2 行：`[3, 5)`

#### 十字链表 / 链式存储
每个非零元素是一个结点，保存：
- data：行号、列号、值
- ptr_row_next:：同行下一个非零元素
- ptr_col_next：同列下一个非零元素
适合：
- 需要频繁插入、删除非零元素
- 动态修改矩阵结构

十字链表：
- 两组头指针数组 + 同一个结点被两套链组织起来
- 二维单向链表

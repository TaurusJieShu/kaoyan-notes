操作特点：
  - 插入：O(1)，已知插入位置时只需修改指针
  - 删除：O(1)，已知待删结点的前驱结点时只需修改指针

  - 遍历：O(n)

		  - 循环访问：元素按顺序重复访问，到末尾后重新回到开头
  - 头结点：链表开始位置附加的结点，通常用于统一操作
  - 虚拟头结点：算法题中常用的辅助结点，用于简化边界处理

	- 虚拟头结点：从 dummy（**哑结点**） 开始遍历，头结点和普通结点统一处理

# 排序的循环链表
> **链表中值最大的节点不指向 null，而是指向前面值最小的节点，形成一个环。**
![324](Pasted%20image%2020260422152619.png)

循环链表特点：
- 至多有1个元素满足 this -> data >= this -> next -> data
- 等式成立条件：所有元素均相等
操作特点：
  - 插入：O(1)，找到 this -> data < current < this -> next -> data 的点插入数据
  - 删除：O(1)，注意处理头尾节点可能不一样
  - 查找：按值查找 O(n)
  - 遍历：O(n)
适用场景：带有排序需求、不需要随机访问的场景，适合处理“反复轮转”的场景，**凡是“转了一圈还要继续”的场景，循环链表就很合适。**
注意事项：如果要判定是否是合规的循环链表
1. 从某个节点开始，遍历一遍数据，至多有1个元素满足 this -> data >= this -> next -> data
2. 在合规的排序循环链表中，遍历到1个元素则能找到最大最小值，如果遍历结果为0，则随便一个元素都是最大最小值
处理模版
```java
// 空链表
if(head == null);
if(head.next == head);

// 遍历
Node cur = head; // 存储head地址
do {
    // 处理 cur
    cur = cur.next; // 每次往后移动一格
} while (cur != head);
```

## 数组与链表常见处理

### 【基础】直接处理：
#### 遍历 O(n)
1. 简述：顺序访问一次数据。
2. 能力：顺序访问，访问时完成**判断、统计、修改或查找**。
3. 要求：“数据 or 结果数据”**没有明显特点**；需要通过遍历一次得到某些特征。
4. 案例：数组求和、统计满足条件的元素个数、链表查找某个值。
#### 其他
- 模拟：直接按题目规则逐步执行。
- 分类讨论：把问题按不同条件拆成若干情况，分别分析和处理


### 【重点】位置协同类：
#### 双指针/快慢指针 O(n)
1. 简述：使用两个指针配合移动处理数据。
2. 能力：通过两个位置之间的关系完成**扫描、夹逼、去重、划分、覆盖、交换**。
3. 要求：“数据 or 结果数据”通常具有**位置关系**；需要边访问边维护左右、前后或快慢位置。
4. 案例：删除有序数组中的重复项、移动零、反转数组、合并有序数组。
	1. 扫描：快慢指针，顺序移动，在遍历过程中**收集信息或完成处理**。【前后信息】
	2. 夹逼：左右指针，两端向中间移动，根据获取**左右信息**逐步缩小范围。【左右信息】
5. 总结双指针和快慢指针：
	1. 双指指针（两侧 / 前后）：两个指针配合移动，重点在于利用**位置关系**处理问题。	
		1. 常用：夹逼从两端向中间收缩 ，前后配合：一个扫描，一个维护结果区 
		2. 双指针（两侧）更常见于**数组**，双指针（前后）也可用于链表
	2. 快慢指针（同侧）：利用**速度差、速度比、固定间距**，维持**可控**和**目标**节点的**相对关系**。
		1. “数据”通常具有**链式结构**，**不能直接随机定位**，需要通过相对位置关系得到答案。

### 【掌握】结构调整类：
#### 指针重连 O(n)：修改局部顺序
1. 简述：通过修改结点之间的指针关系完成操作。
2. 能力：完成链表中的**插入、删除、拼接、反转、重排**等结构调整。
3. 要求：“数据 or 结果数据”通常是**链表**；重点不在改值，而在修改连接关系。
4. 案例：删除链表结点、反转链表、两两交换链表结点、链表重排。
#### 反转 O(n)：修改前后顺序
1. 简述：将数据顺序整体或局部翻转。
2. 能力：改变元素或结点的**前后顺序**；数组中多用交换，链表中多用改指针方向。
3. 要求：“数据 or 结果数据”通常存在**逆序需求**，或需要把局部顺序翻转。
4. 案例：反转字符串、反转链表、局部反转链表。
#### 合并 O(n)：完成结构合并
1. 简述：将两个或多个结构按规则组合成一个结果。
2. 能力：通过比较或拼接完成**有序合并、结果构造、整体整合**。
3. 要求：“数据 or 结果数据”通常已有**一定规则或顺序**，合并时需要保持该规则。
4. 案例：合并两个有序数组、合并两个有序链表。

> 合并有序链表（【推荐】递归写法）
```c
Node* merge(Node* l1, Node* l2) {
    if (l1 == NULL) return l2;
    if (l2 == NULL) return l1;

    if (l1->data < l2->data) {
        l1->next = merge(l1->next, l2);
        return l1;
    } else {
        l2->next = merge(l1, l2->next);
        return l2;
    }
}
```

## 1. 长度最小的子数组
给定一个含有 `n` 个正整数的数组 `nums` 和一个正整数 `target`。  
找出该数组中满足其和 `>= target` 的长度最小的连续子数组，并返回其长度；如果不存在这样的子数组，返回 `0`。na

> 题目复述：在数组中找到一个窗口内满足元素数量最小，同时窗口值和>=target。
> 元素：数组、连续、窗口、目标值、遍历
> 思考：连续 → 同侧双指针。目标值/元素和 → 滑动窗口。
> 思路：右指针找到第一个满足 和>=target 的地址，right每次向右扩张1格，然后left都要进行向左收缩到最小满足条件的时刻，每次收缩完保存数据。
> 复杂度：时间复杂度 O(n) 空间复杂度 O(1)

## 2. 盛最多水的容器
给定一个长度为 `n` 的整数数组 `height`。  
有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])``。  
找出其中两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

> 题目复述：找出数组中，$S_{水量} = D_{(a-b)}*\min(a,b)$，导致 $S_{水量}$  最大的一组数。
> 元素：数组、考虑到距离 → 两侧双指针 + 两侧保存点 + 最大水量，贪心剪枝算法。
> 思考：
> 1. 由于要考虑下标距离，如果从同侧我们既要考虑距离又要考虑高度，所以我们固定从两侧开始看，这样下标距离就都是递减的，比较好思考。→ 两侧双指针
> 2. 选定左右柱子低的进行移动，然后进行判断接水量如果接水量高于当前就保存，然后继续判断，如果不是更高就继续循环
> 3. 【错误分析】这里注意我曾经想的是，取接下来的一个柱子高一侧进行移动，但是在间隔缩小的情况下，但凡要是如果移动高的那个柱子，一定是得不到最优解的，所以要移动低的那一边
> 复杂度：时间复杂度 O(n) 空间复杂度 O(1)

## 3. 三数之和
给你一个整数数组 `nums`，判断是否存在三元组 `[a,b,c]`，使得 `a+b+c=0`。  
请找出所有不重复的三元组。

> 题目分析：找出数组里的所有搭配中，存在三个数组结果=0，保存所有搭配，去重。
> 元素：数组、所有组合、结果为0 、去重
> 思考：
> 1. 所有组合 = $C_n^3 = \frac{n!}{3!(n-3)!}$  ，如何利用其他条件进行剪枝？
> 2. 既然当前的暴力解法是$O(n^3)$，而且想不到其他的解法 → 排序，快排 $O(n\log_2n)$
> 3. 那既然数据有序了，那我从头开始选一个数 → 然后从两侧夹逼，得到第一对数字。
> 4. 两侧夹逼的逻辑是，从两侧开始找，如果结果大了右边界缩小，如果小了左边界缩小。
> 5. 由于两数之和为X的有多对，所以要遍历所有的内容让满足 $left < right$ 才能继续遍历。
> 6. 找到第一对之后需要把左右都向中间缩，保证在中间继续找（并且只有中间还有）
> 7. 得到所有的组别，把链表解出来返回值并销毁

```c
/**
 * Return an array of arrays of size *returnSize.
 * The sizes of the arrays are returned as *returnColumnSizes array.
 * Note: Both returned array and *columnSizes array must be malloced, assume caller calls free().
 */
void quick_sort_inplace(int* nums, int numsSize) {
	if(nums == NULL || numsSize == 1 || numsSize == 0) return;
	int i = 0;
	int j = numsSize - 1;
	int pivot = nums[0];
	while (i < j) {
		while (i < j && nums[j] >= pivot) j --;
		nums[i] = nums[j];
		while (i < j && nums[i] <= pivot) i ++;
		nums[j] = nums[i];
	}
	nums[i] = pivot;
	quick_sort_inplace(nums, i);
	// 递归左边
	quick_sort_inplace(nums + i + 1, numsSize - 1 - i);
	// 递归右边
}
typedef struct Node {

MiniNode;

	current->array[0] = a;
	current->array[1] = b;
	current->array[2] = c;

	head->next = current;
}
void destory_head(MiniNode* head) {
	if(head == NULL) return;
	destory_head(head->next);
	free(head);
}
int** threeSum(int* nums, int numsSize, int* returnSize, int** returnColumnSizes) {
	if(numsSize < 3 || nums == NULL) {
		*returnSize = 0;
		*returnColumnSizes == NULL;
		return NULL;
	}
	quick_sort_inplace(nums, numsSize);
	MiniNode head;
	head.next = NULL;
	int head_size = 0;
	for (int i = 0; i < numsSize; i++) {
		int target = nums[i];
		// 当前目标值
		if (i > 0 && nums[i] == nums[i - 1]) continue;
		// 由于数据有序所有双指针两侧夹逼得到第一组数据，但需要满足 left < right
		int left = i+1, right = numsSize - 1;
		// 从两侧开始
		while (left < right) {
			if (nums[left] + nums[right] + target == 0) {
				// 记录该值
				insert_node(&head, target, nums[left], nums[right]);
				head_size ++;
				while (left < right && nums[left] == nums[left + 1]) left++;
				while (left < right && nums[right] == nums[right - 1]) right--;
				left++;
				right--;
			} else if (nums[left] + nums[right] + target < 0) {
				// 比目标值小 → 把值变大
				left ++;
			} else {
				// 比目标值大 → 把值变小
				right --;
			}
		}
	}
	// 到此获取到所有的元素在 head 链表里, 数量为head_size
	*returnColumnSizes = malloc(sizeof(int) * head_size);
	for (int i = 0; i < head_size; i++) {
		(*returnColumnSizes)[i] = 3;
	}
	
	MiniNode *current = head.next;
	int** result = malloc(sizeof(int*) * head_size);
	for (int i = 0; i < head_size; i++) {
		result[i] = malloc(sizeof(int) * 3);
		result[i][0] = current->array[0];
		result[i][1] = current->array[1];
		result[i][2] = current->array[2];
		current = current -> next;
	}
	// 返回数据
	*returnSize = head_size;
	// 释放head
	destory_head(head.next);
	return result;
}
```

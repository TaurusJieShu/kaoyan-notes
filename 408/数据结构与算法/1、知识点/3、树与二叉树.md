# 树与二叉树

## 树
**树的概念**：
1. **空树**：节点数为0的树。
2. **度**：
	1. **结点的度**：该结点拥有的子树个数。
	2. **树的度**：树中各结点度的**最大值**。
3. **结点**：
	1. **叶子结点**：度为$0$的结点，也称终端结点。
	2. **分支结点**：度大于$0$的结点，也称非终端结点。
	3. **孩子结点**：某结点子树的根结点。
	4. **双亲结点**：某结点的直接前驱。
	5. **兄弟结点**：同一双亲的孩子结点之间互称兄弟。
	6. **祖先结点**：从根到该结点路径上的所有结点。
	7. **子孙结点**：某结点的子树中的任意结点。
4. **整体**：
	1. **层次**：根结点为第$1$层，其余结点层次等于双亲结点层次加$1$。
	2. **深度（高度）**：树中结点的最大层数。
5. **场景差别**：
	1. **有序树**：子树间有次序，不能互换。（语法树、二叉树、表达式树）
	2. **无序树**：子树间无次序，可以互换。（组织架构等）
6. **森林**：$m(m \ge 0)$棵互不相交的树的集合。

**树的性质**：
1. 【树的结点个数】树中所有节点的度 + 1(根) = 结点个数
2. 【树的结点个数】高度为h，度为m的树，最多有 $\frac{q^n - 1}{q - 1}$个元素（等比数列求和公式）
3. 【第i层的结点个数】度为m的树(m叉树)，第i层最多有$m^{i-1}$个结点
4. 【高度】具有n个结点的m叉树的最小高度为
	1. 根据等比数列求和公式  $n = 1 · \frac{m^h - 1}{m - 1}$ 向上取整

## 二叉树
定义：度为2的有序树，
**二叉树的性质**：
1. 【树的结点个数】树中所有节点的度 + 1(根) = 结点个数【与树一致】
2. 【树的结点个数】高度(深度)为h，度为2的树，最多有 $2^n - 1$个元素【树的特例】
3. 【第i层的结点个数】度为2的树，第i层最多有$2^{i-1}$个结点 【树的特例】
4. 【高度】具有n个结点的m叉树的最小高度为
	1. 根据等比数列求和公式  $n = 2^h - 1$ 向上取整
5. 【度】对任何非空二叉树，若叶子结点数为$n_0$，度为$1$的结点数为$n_1$，度为$2$的结点数为$n_2$
	1. $0 · n_0 + 1 · n_1 + 2·n_2 + 1 = n$ → $n - n_1 = n_2 + n_0 = 2n_2 + 1$
	2. $n_0 = n_2 + 1$
	3. 把树想象成一条水流，如果某个点只分出 1 条路，那出口数不变。如果某个点分出 2 条路，那出口数就比原来多 1 个。
6. 常见形态：
	1. **完全二叉树**：除了最后一层，每一层都是满的
		1. 定义：深度为$k$的二叉树，除第$k$层外，其余各层都达到最大结点数，且第$k$层的结点都连续集中在最左边。
		2. 特点：
			1. 适合顺序存储。
			2. 编号具有连续性。
		3. 特例：**满二叉树**
		4. 对于具有$n$个结点的完全二叉树，若按层序编号为$1 \sim n$，则对编号为$i$的结点：
			1. 【父亲结点】若$i > 1$，其双亲结点编号为$\lfloor i/2 \rfloor$
			2. 【左子结点】若$2i \le n$，其左孩子编号为$2i$
			3. 【右子结点】若$2i+1 \le n$，其右孩子编号为$2i+1$
			4. 【叶子结点】若$i > \lfloor n/2 \rfloor$，则编号为$i$的结点为叶子结点。
			5. 【叶子结点】完全二叉树中，叶子结点只可能出现在最后两层。
7. **斜树**：退化成链表
	1. 所有结点都只有左子树，称左斜树。
	2. 所有结点都只有右子树，称右斜树。

# 二叉树的遍历

**先/中/后 序遍历**：
1. 先序遍历（PreOrder）：根 $\rightarrow$ 左 $\rightarrow$ 右
2. 中序遍历（InOrder）：左 $\rightarrow$ 根 $\rightarrow$ 右
3. **后序遍历（PostOrder）**：左 $\rightarrow$ 右 $\rightarrow$ 根
实现方式：
- **递归实现**：对当前结点按“访问根、左子树、右子树”的不同次序进行递归处理。
- **非递归实现**：
	1. 先序遍历：借助栈，访问结点后先压右孩子再压左孩子。
	2. 中序遍历：借助栈，一路向左入栈，出栈访问后转向右子树。
	3. 后序遍历：借助栈和访问标记，或用双栈实现“左右根”顺序。


```c
typedef struct BiTNode {
	int data;
	struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;
```

- **非递归实现**：

1. **先序遍历**：借助栈，先访问当前结点，再将右孩子、左孩子依次入栈，使左子树先出栈访问。
```c
void PreOrder(BiTree T) {
	if (T == NULL) return;

	BiTree stack[1000];
	int top = -1;
	stack[++top] = T;

	while (top != -1) {
		BiTree p = stack[top--];
		printf("%d ", p->data);

		if (p->rchild) stack[++top] = p->rchild;
		if (p->lchild) stack[++top] = p->lchild;
	}
}
```

2. **中序遍历**：借助栈，沿左链不断入栈，直到为空后出栈访问，再转向该结点的右子树继续重复。
```c
void InOrder(BiTree T) {
	BiTree stack[1000];
	int top = -1;
	BiTree p = T;

	while (p != NULL || top != -1) {
		while (p != NULL) {
			stack[++top] = p;
			p = p->lchild;
		}

		p = stack[top--];
		printf("%d ", p->data);
		p = p->rchild;
	}
}
```

3. **后序遍历**：借助栈，先沿左链入栈，再根据右孩子是否已访问决定是转向右子树还是访问当前结点；也可用双栈先得到“根右左”，再逆序为“左右根”。
### 方法一：单栈 + 最近访问结点
```c
void PostOrder(BiTree T) {
	BiTree stack[1000];
	int top = -1;
	BiTree p = T;
	BiTree lastVisited = NULL;

	while (p != NULL || top != -1) {
		while (p != NULL) {
			stack[++top] = p;
			p = p->lchild;
		}

		BiTree node = stack[top];
		if (node->rchild != NULL && node->rchild != lastVisited) {
			p = node->rchild;
		} else {
			printf("%d ", node->data);
			lastVisited = node;
			top--;
		}
	}
}
```

### 方法二：双栈（推荐）栈1依次入栈，然后适时弹出栈保存
```c
void PostOrder2(BiTree T) {
	if (T == NULL) return;

	BiTree s1[1000], s2[1000];
	int top1 = -1, top2 = -1;
	s1[++top1] = T;

	while (top1 != -1) {
		BiTree p = s1[top1--];
		s2[++top2] = p;

		if (p->lchild) s1[++top1] = p->lchild;
		if (p->rchild) s1[++top1] = p->rchild;
	}

	while (top2 != -1) {
		printf("%d ", s2[top2--]->data);
	}
}
```

**层序遍历（LevelOrder）**：
	1. 顺序：按层从上到下、从左到右
	2. 一句话简述：借助队列，根结点先入队，每次出队一个结点并将其左右孩子依次入队。

# 前中后序遍历画树技巧总结
**已知两种遍历时能否唯一确定**：
1. **前序 + 中序**：**可以唯一确定**
2. **后序 + 中序**：**可以唯一确定**
3. **前序 + 后序**：**一般不能唯一确定**（满二叉树中每个非叶子结点都有两个孩子，不会出现“单孩子结点左右不确定”的问题）

**前序 + 中序画树技巧**：
1. 前序遍历第一个结点（**根结点**） →  中序遍历的根结点，分左右
	1. 根结点左边部分是**左子树的中序**：根据中序左子树结点数n = 根后前n个是**左子树前序**
	2. 根结点右边部分是**右子树的中序**：剩下的是**右子树前序**
2. 对左右子树递归重复以上过程

**后序 + 中序画树技巧**：
1. 后序遍历最后一个结点（**根结点**）→ 中序遍历的根结点，分左右
	1. 根结点左边部分是**左子树的中序**：根据中序左子树结点数n = 根前n个是**左子树后序**
	2. 根结点右边部分是**右子树的中序**：再后若干个是**右子树后序**
2. 对左右子树递归重复以上过程

**和卡特兰数有关**：
1. 不同二叉树个数是典型的卡特兰数问题【卡特兰数的计算：$C_n = \frac{1}{n+1}C_{2n}^{n}$】
2. 因为二叉树具有递归划分结构：
	1. 先确定根结点
	2. 剩余结点分配给左子树和右子树
	3. 左右子树又分别是同类问题
3. 若左子树有$i$个结点，则右子树有$n-1-i$个结点
4. 当$n=4$时：$C_4 = \frac{1}{5}\binom{8}{4} = 14$
5. **为什么和前序遍历$a,b,c,d$无关**
	1. 题目虽然给了前序遍历为$a,b,c,d$，前序遍历**不能唯一确定**二叉树结构
	2. 对于任意一种$4$结点二叉树，都可以按其前序访问顺序，把结点依次标记为：a, b, c, d
	3. 所以“前序遍历为$a,b,c,d$”并没有额外限制树形，只是规定了结点的命名顺序
	4. 因而题目本质仍然是：$4$个结点的不同二叉树个数
